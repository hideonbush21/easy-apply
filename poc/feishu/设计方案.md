# 飞书机器人 POC 设计方案

## 1. 目标

在 `poc/feishu` 下搭建独立 Flask 服务，实现完整的飞书机器人消息交互闭环：

用户在飞书群里 @机器人发消息 → 后端接收 → LLM 处理 → 回复到群聊

## 2. 技术选型

| 组件 | 选择 | 理由 |
|------|------|------|
| SDK | `lark-oapi` (飞书官方 Python SDK) | 封装鉴权、事件解析、消息发送，提供 Flask adapter |
| Web 框架 | Flask（独立实例） | 与主项目技术栈一致，POC 阶段不耦合主后端 |
| 异步处理 | `threading.Thread` | 飞书要求 3s 内返回 HTTP 200，LLM 调用放后台线程 |
| 内网穿透 | ngrok | 本地开发阶段将服务暴露到公网 |
| LLM | OpenAI Chat API | 项目已有 OPENAI_API_KEY，直接复用 |
| 端口 | 9999 | 避免与主后端 5001 冲突 |

## 4. 核心流程

```
飞书服务器 POST /webhook/event
          │
          ├─ challenge 请求？
          │    └─ 返回 {"challenge": "xxx"}（URL 验证，仅首次配置时触发）
          │
          └─ im.message.receive_v1 消息事件？
               │
               ├─ 事件去重检查（event_id 是否已处理过）
               │    └─ 重复事件 → 直接返回 200，不处理
               │
               ├─ 立即返回 HTTP 200（满足飞书 3 秒超时限制）
               │
               └─ 启动后台线程：
                    1. 解析用户消息文本（去掉 @机器人 的 mention 部分）
                    2. 调用 OpenAI Chat API 生成回复
                    3. 通过飞书 API reply 到原消息（群聊可见）
```

## 5. 模块设计

### 5.1 `app.py` — Flask 应用 + Webhook 端点

**职责**：接收飞书事件推送，分发给处理函数

**关键实现**：
- 使用 `lark-oapi` 的 `EventDispatcherHandler` 注册事件回调
- 通过 Flask adapter（`parse_req` / `parse_resp`）桥接 Flask 和 SDK
- 注册 `p2_im_message_receive_v1` 事件处理函数

**代码结构**：
```python
import lark_oapi as lark
from lark_oapi.adapter.flask import parse_req, parse_resp

# 1. 创建 lark client（用于主动调 API）
client = lark.Client.builder()
    .app_id(FEISHU_APP_ID)
    .app_secret(FEISHU_APP_SECRET)
    .build()

# 2. 创建事件处理器
handler = lark.EventDispatcherHandler.builder(ENCRYPT_KEY, VERIFICATION_TOKEN)
    .register_p2_im_message_receive_v1(on_message_receive)
    .build()

# 3. Flask 路由
@app.route("/webhook/event", methods=["POST"])
def webhook_event():
    resp = handler.do(parse_req())
    return parse_resp(resp)
```

### 5.2 `bot_handler.py` — 消息处理逻辑

**职责**：解析消息 → 调用 LLM → 回复飞书

**核心函数**：

#### `on_message_receive(data)`
- 事件回调入口
- 提取 message_id、chat_id、消息内容、发送者信息
- 事件去重（基于 event_id，使用内存 set + TTL 淘汰）
- 启动后台线程调用 `handle_and_reply()`

#### `handle_and_reply(client, message_id, user_text)`
- 调用 OpenAI Chat API，system prompt 可自定义
- 通过 `client.im.v1.message.reply()` 回复到原消息
- 错误处理：LLM 调用失败时回复友好提示

#### `clean_mention_text(content, mentions)`
- 飞书消息中 @机器人 的文本格式为 `@_user_1 你好`
- 需要去掉 mention 占位符，提取纯用户输入

## 6. 飞书 API 交互细节

### 6.1 事件数据结构（v2.0 schema）

```json
{
    "schema": "2.0",
    "header": {
        "event_id": "unique_event_id",
        "event_type": "im.message.receive_v1",
        "token": "verification_token",
        "app_id": "cli_xxx"
    },
    "event": {
        "sender": {
            "sender_id": {
                "open_id": "ou_xxx",
                "union_id": "on_xxx"
            },
            "sender_type": "user"
        },
        "message": {
            "message_id": "om_xxx",
            "chat_id": "oc_xxx",
            "chat_type": "group",
            "message_type": "text",
            "content": "{\"text\":\"@_user_1 你好\"}",
            "mentions": [
                {
                    "key": "@_user_1",
                    "id": { "open_id": "ou_bot_xxx" },
                    "name": "机器人名称"
                }
            ]
        }
    }
}
```

### 6.2 回复消息 API

```
POST https://open.feishu.cn/open-apis/im/v1/messages/{message_id}/reply
Authorization: Bearer {tenant_access_token}

{
    "content": "{\"text\":\"这是机器人的回复\"}",
    "msg_type": "text"
}
```

SDK 封装后：
```python
from lark_oapi.api.im.v1 import ReplyMessageRequest, ReplyMessageRequestBody

request = ReplyMessageRequest.builder()
    .message_id(message_id)
    .request_body(ReplyMessageRequestBody.builder()
        .content(json.dumps({"text": reply_text}))
        .msg_type("text")
        .build())
    .build()

client.im.v1.message.reply(request)
```

### 6.3 Token 管理

`lark-oapi` SDK 自动处理 `tenant_access_token` 的获取和刷新，无需手动管理。

## 7. 事件去重策略

飞书在未收到 200 响应时会重试推送（最多 3 次），需要防止重复处理。

**方案**：内存 set + 过期淘汰
```python
# 使用 dict 存储 event_id -> timestamp
# 每次处理前检查是否已存在
# 定期清理超过 5 分钟的旧 event_id
```

POC 阶段用内存即可，生产环境可改用 Redis。

## 9. 飞书开放平台配置清单

| 权限 | 说明 |

| `im:message` | 获取与发送单聊、群组消息 |

## 10. 本地开发与测试步骤

```bash

pip install -r requirements.txt

# 2. 配置环境变量
```
